Methods


    int addNums(int numOne, int numTwo)
    {
        return numOne + numTwo;
    }

Methods, sometimes referred to as class members, are chunks of reusable code. In C#, they exist within our classes to give us functionality we can call off our objects to get some sort of desired effect. They are capable of taking in data as parameters, and can also return data back out of them. In their simplest forms, methods are really just collections of statements, executed in a series, that we can reuse whenever needed throughout our programs.

Classes


    class Cat 
    {
        public int Age { get; set; }
        public string Name { get; set; }
        public bool IsCute { get; set; }

        static string Meow()
        {
            return "Meow";
        }
    }

A class is a kind of blueprint for objects containing both methods, and properties. They decide what kind of data our objects with have with properties, and lay out what kind of actions our objects can take with methods. Each object constructed from a class will have the same properties and methods available, but they may have different values contained in those properties.

Lists


    List<Cat> catList = new List<Cat>();
    Cat quin = new Cat();
    Cat snowball = new Cat();

    catList.Add(quin);
    catList.Add(snowball);

C# has many different types of collections, but a very important one is the List<>. A List<> is really just a list of values of the same data type, that you can continue to add to as need be through prebuilt methods of the List<> class. Remember that these collections have a generic type by default, and require us to give them a type in between the angle brackets. We can even use classes we define as types of List<>!
